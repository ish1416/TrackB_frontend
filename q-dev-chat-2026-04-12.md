## Backend Bug Report — Upload Stuck at 95% (Large APK Timeout)

### What's Happening

The progress bar hits 95% and freezes because the backend **never sends a final HTTP response** for large APKs.

**Evidence from live testing:**

| APK | Size | Result |
|---|---|---|
| ASR YellowSense | 4.6 MB | ✅ Returns in 9 seconds |
| AU Bank | 159 MB | ❌ Hangs forever — times out after 120s |

**The exact failure sequence:**
```
Frontend sends:  POST /api/v1/analyze/unified  (159 MB file)
Backend replies: HTTP 100 Continue             ← "I got your headers, send the body"
Frontend sends:  File body (159 MB)
Backend:         [SILENCE — never sends 200/413/504]
Frontend:        Progress bar stuck at 95% forever
```

`SIZE_UPLOAD: 26,969,148` — only **27 MB of the 159 MB** was uploaded before the connection died. The backend is **closing the connection mid-upload**.

### Root Causes (Two separate issues)

**Issue 1 — Uvicorn/FastAPI request body size limit**

FastAPI/Uvicorn has a default `max_request_body_size` that is likely set too low. When the body exceeds it, Uvicorn drops the connection without sending a proper error response — it just closes the TCP socket, which the frontend sees as a hang.

**Fix for backend team:**
```python
# In uvicorn startup command:
uvicorn main:app --host 0.0.0.0 --port 8000 --limit-max-requests 0 --timeout-keep-alive 120

# OR in FastAPI app initialization:
from uvicorn.middleware.proxy_headers import ProxyHeadersMiddleware
app.add_middleware(ProxyHeadersMiddleware, trusted_hosts="*")
```

Or if using a `uvicorn.run()` call:
```python
uvicorn.run(app, host="0.0.0.0", port=8000, 
            limit_concurrency=10,
            timeout_keep_alive=120,
            h11_max_incomplete_event_size=200 * 1024 * 1024)  # 200MB
```

**Issue 2 — Nginx `client_max_body_size` (already known)**

```nginx
# /etc/nginx/conf.d/upload_limit.conf
client_max_body_size 200M;
client_body_timeout 120s;
proxy_read_timeout 120s;
proxy_send_timeout 120s;
```

Run on VM:
```bash
echo -e "client_max_body_size 200M;\nclient_body_timeout 120s;\nproxy_read_timeout 120s;\nproxy_send_timeout 120s;" | sudo tee /etc/nginx/conf.d/upload_limit.conf
sudo nginx -t && sudo systemctl reload nginx
```

**Issue 3 — No proper error response on oversized files**

When the file is too large, the backend should return `413 Payload Too Large` immediately instead of silently dropping the connection. This is what causes the frontend to hang — it's waiting for a response that never comes.

**Fix:**
```python
from fastapi import Request, HTTPException

@app.middleware("http")
async def limit_upload_size(request: Request, call_next):
    max_size = 200 * 1024 * 1024  # 200 MB
    if request.headers.get("content-length"):
        if int(request.headers["content-length"]) > max_size:
            raise HTTPException(status_code=413, detail="File too large. Maximum size is 200MB.")
    return await call_next(request)
```
